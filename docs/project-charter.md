# Project Charter

## Vision

Build a production-grade DevOps platform from scratch to master Infrastructure as Code, automation, containers, Kubernetes, CI/CD and GitOps. The platform will serve as my learning environment and professional portfolio.

## Mission

Design, build, automate and continuously improve a production-like platform using industry best practices. Every component will be reproducible, documented and managed as code.

## Goals

- Master Linux fundamentals
- Master Git and GitHub
- Master Terraform
- Master Ansible
- Master Docker
- Master Kubernetes
- Master CI/CD
- Master GitOps
- Master Monitoring
- Achieve AZ-104 certification
- Achieve AZ-400 certification
- Build a professional GitHub portfolio
- Become job-ready as a DevOps Engineer within 12 months

## Non-Goals

- Chasing certifications without practical experience
- Building unnecessary complexity
- Using tools without understanding them
- Skipping documentation
- Manual configuration when automation is possible
- Vendor lock-in where avoidable

## Engineering Principles

- Automation First
- Infrastructure as Code
- Security by Default
- Simplicity over Complexity
- Documentation First
- Git as the Single Source of Truth
- Version Everything
- Fail Fast, Learn Fast
- Reproducibility over Manual Work
- Continuous Improvement
